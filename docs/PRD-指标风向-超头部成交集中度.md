# PRD：指标风向 — 超头部成交集中度

> 文档状态：草案（待评审）  
> 适用范围：观澜 A 股短线雷达（GuanlanRadar）  
> 作者：Kimi Code CLI  
> 最后更新：2026-06-28

## 1. 背景与目标

### 1.1 背景

当前观澜已具备「全景雷达」「板块 Top10/Bottom10」「股票详情」「股票推荐」「我的持股」「个股讨论」「虚拟交易」等模块，能够从指数、板块、个股、持仓、AI 讨论等角度辅助短线决策。但在判断「市场资金是否抱团头部」「抱团程度处于什么历史分位」「抱团板块集中在哪」「极端抱团后行情可能如何演变」等宏观择时问题上，尚缺少一个结构化、可量化的独立入口。

### 1.2 目标

新增一级菜单 **「指标风向」**，首期聚焦 **「超头部成交集中度」** 指标：

- 在沪深 A 股全市场范围内，按**成交额**排序，从 **Top25、Top1%、Top5%** 三个维度计算头部股票成交额占全市场成交额的比重。
- 展示**当前数值、历史趋势、分位等级、板块分布、资金抱团趋势**。
- 结合历史规律与当前市场环境，输出**行情走势变化可能性**的辅助判断。
- 与观澜现有数据体系（行情源、缓存、AI 分析、自动刷新、快照兜底）保持一致，优先使用已有接口与存储机制。

## 2. 名词定义

| 术语 | 定义 |
|------|------|
| 指标风向 | 新增的一级功能菜单入口，用于展示单一或多个宏观择时指标。 |
| 超头部成交集中度 | 以成交额衡量的头部股票交易占比，反映市场资金向极少数股票的集中程度。 |
| 沪深股市范围 | 上海证券交易所与深圳证券交易所全部 A 股（含主板、科创板、创业板、北交所是否纳入需在第 4.1 节确认）。默认不含 B 股、港股通、基金、债券。 |
| 成交额 | 个股当日成交金额（人民币，单位：元）。计算口径与行情源保持一致，优先取 `tencent` 或 `eastmoney` 实时行情的 `amount`/`成交额` 字段。 |
| Top25 | 按当日成交额从高到低排序，取前 25 只股票。 |
| Top1% | 按当日成交额从高到低排序，取前 `ceil(N × 1%)` 只股票，N 为当日有效样本总数。 |
| Top5% | 按当日成交额从高到低排序，取前 `ceil(N × 5%)` 只股票。 |
| 集中度 | `头部股票成交额合计 / 全市场样本成交额合计 × 100%`。 |
| 资金抱团 | 资金持续向少数头部股票或少数板块聚集的市场状态。 |
| 行情走势变化可能性 | 基于当前集中度等级、趋势方向、板块分布及近期市场特征，给出的定性概率判断（如「高位抱团松动概率上升」）。 |

## 3. 功能范围

### 3.1 In Scope

1. 新增「指标风向」菜单入口与独立页面。
2. 「超头部成交集中度」卡片/仪表盘的当前数据展示。
3. 三个维度（Top25 / Top1% / Top5%）的集中度实时计算。
4. 历史趋势曲线（至少 60 个交易日，目标 120 个交易日）。
5. 头部股票列表及所属板块/行业分布。
6. 板块集中度排行与占比可视化。
7. 资金抱团趋势标签（如「上升/下降/平稳」「极端/高位/正常/低位」）。
8. 行情走势变化可能性分析文本，支持 AI 生成与规则模板兜底。
9. 数据缓存、快照、自动刷新、手动刷新机制与现有观澜体系一致。
10. 移动端适配与现有 UI 风格保持一致。

### 3.2 Out of Scope（首期）

1. 用户自定义集中度阈值或维度。
2. 按流通市值、换手率、主力净额等其他口径的集中度。
3. 与其他市场（港股、美股）的横向对比。
4. 基于该指标的信号提醒或自动化交易。
5. 分钟级/日内实时集中度曲线（首期以日级为主）。

## 4. 数据需求

### 4.1 样本范围

- **默认范围**：沪深 A 股全部上市股票（以行情源返回的 A 股代码列表为准）。
- **代码前缀**：包含 `60`、`00`、`30`、`68`、`8`、`4`（北交所/新三板可视数据质量决定是否纳入，建议首期默认排除，页面提供开关）。
- **排除项**：停牌无成交额、ST/*ST（可选开关，默认不排除，但在分析文本中提示风险）。
- **样本总数 N**：以当前行情源能获取到有效成交额的股票数量为准，不是交易所挂牌总数。

### 4.2 计算口径

| 维度 | 计算方式 | 备注 |
|------|---------|------|
| Top25 | 成交额第 1～25 名合计 / 全样本成交额合计 | 固定数量，适合横向比较 |
| Top1% | 成交额前 `ceil(N × 1%)` 名合计 / 全样本成交额合计 | 随市场扩容缓慢变化 |
| Top5% | 成交额前 `ceil(N × 5%)` 名合计 / 全样本成交额合计 | 反映广义头部集中度 |

- 每日收盘后或盘中刷新时，使用最新行情重新计算。
- 历史趋势数据以**日终收盘价对应的成交额**为准，避免盘中波动导致历史曲线口径不一致。

### 4.3 板块/行业映射

- 优先使用东方财富行业分类（与现有「板块」模块一致）。
- 若行情源未返回行业，则尝试通过本地映射表或 `eastmoney` 板块接口补齐。
- 头部股票板块分布 = 头部股票成交额按所属行业汇总 / 头部股票成交额合计。

### 4.4 历史数据

- 回溯窗口：默认 60 个交易日，支持切换 20/60/120 日。
- 存储位置：`data/concentration-history.json`（新增）。
- 更新策略：
  - 每个交易日收盘后（15:05 后）自动追加一条日终记录。
  - 若当天数据缺失，启动时尝试从行情源补全最近 N 个交易日。
  - 最多保留 250 个交易日，超出按 FIFO 清理。

### 4.5 历史数据缓存

为避免每次打开页面或重启服务都重新采集、重新计算历史集中度，新增**两级历史数据缓存机制**：磁盘归档缓存 + 内存热缓存。

#### 4.5.1 缓存目标

- **避免重复采集**：历史交易日的全市场成交额只需采集一次，后续直接读取已归档的日终数据。
- **避免重复计算**：历史集中度、分位、板块分布等衍生结果一旦生成，按日期缓存，后续直接复用。
- **加速页面首屏**：打开「指标风向」页面时，历史趋势图优先从缓存渲染，同时在后台异步校验/补齐最新数据。

#### 4.5.2 缓存层级

| 层级 | 存储位置 | 有效期 | 用途 |
|------|----------|--------|------|
| L1 内存热缓存 | `ConcentrationHistoryCache`（Node.js 进程内存） | 交易时段 5 分钟，非交易时段 60 分钟 | 接口 `/api/indicator/concentration/history` 快速响应 |
| L2 磁盘归档缓存 | `data/concentration-history.json` | 永久（最多保留 250 个交易日） | 跨重启持久化，缓存未命中或进程重启后恢复 |
| L3 源数据快照 | `data/market-snapshot.json`（已有） | 与现有快照策略一致 | 所有实时源失败时的兜底 |

#### 4.5.3 缓存内容

每条缓存记录至少包含：

```jsonc
{
  "date": "2026-06-27",
  "scope": "沪深A股",
  "sampleCount": 5300,
  "dataSource": "eastmoney",
  "dimensions": {
    "top25": { "ratio": 12.85, "amount": 185000000000 },
    "top1pct": { "ratio": 18.42, "amount": 265000000000 },
    "top5pct": { "ratio": 35.17, "amount": 506000000000 }
  },
  "totalAmount": 1440000000000,
  "industryDistribution": [
    { "industry": "半导体", "amount": 4500000000, "ratio": 35.2 }
  ],
  "checksum": "sha256-of-raw-snapshot-or-quote-hash", // 用于校验源数据是否变化
  "cachedAt": "2026-06-27T15:06:00+08:00"
}
```

#### 4.5.4 缓存更新策略（增量而非全量）

1. **启动恢复**：服务启动时读取 `data/concentration-history.json`，加载最近 250 个交易日到 L1 内存缓存。
2. **每日追加**：交易日 15:05 归档任务仅计算并追加**当天**记录，不对全部历史重新计算。
3. **缺失补全**：若发现最近 N 个交易日（如周末、节假日后）存在缺失：
   - 优先尝试从行情源补采缺失日期的日终行情；
   - 若行情源无法提供历史日终数据，则保留缓存中的空档，页面提示「历史数据存在缺失」。
4. **变更重算**：仅当以下情况触发**指定日期**的重算，而非全量重建：
   - 该日期的源数据快照发生变更（通过 `checksum` 检测）；
   - 样本范围或计算口径配置变更；
   - 用户手动触发「刷新历史数据」。

#### 4.5.5 缓存命中与失效规则

- **命中条件**：请求窗口内所有交易日均有缓存记录，且 `scope`、`dimensions` 版本号一致。
- **过期条件**：
  - 内存缓存超过 TTL（交易时段 5 分钟 / 非交易时段 60 分钟）；
  - 当日收盘后生成新的日终记录；
  - 检测到计算口径或样本范围配置变更。
- **失效粒度**：按日期失效，不影响其他日期的缓存。

#### 4.5.6 缓存预热

- 服务启动完成后，后台异步执行一次「历史数据完整性检查」，对缺失的最近 5 个交易日尝试补采。
- 交易时段内，每次页面打开后若缓存未过期，直接返回；若过期，先返回旧缓存，后台更新后再推送增量。

#### 4.5.7 降级与容灾

- 行情源历史数据不可用：使用磁盘缓存中的数据，即使存在几天的空档也继续展示。
- 磁盘缓存损坏或为空：
  - 启动时尝试从 `data/market-snapshot.json` 恢复最近 1 条记录；
  - 页面提示「历史数据积累中，趋势图暂不可用」，并开始新的归档积累。
- 计算异常：单条日期计算失败时仅跳过该日期，不影响整体缓存。

### 4.6 数据源优先级

与现有设置页「行情数据源」保持一致：

1. `auto`：优先 `eastmoney`（A 股全列表+成交额），失败则 fallback 到 `tencent`。
2. `tencent`：通过腾讯行情批量获取全 A 股实时行情。
3. `eastmoney`：通过东方财富接口获取全 A 股列表与成交额。
4. `sina`：作为最后兜底。

所有实时源失败时，使用 `data/market-snapshot.json` 中最近一次有效快照作为降级展示，并显式提示「数据非实时」。

## 5. 产品功能详细说明

### 5.1 入口与导航

- 在主导航栏（与「全景雷达」「股票推荐」「我的持股」等同级）新增 **「指标风向」** 入口。
- 图标建议：风向标或雷达波样式，与观澜主视觉一致。
- 未读/提示：首期不做红点提醒。

### 5.2 页面布局

页面采用从上到下的信息流：

```
┌─────────────────────────────────────────────┐
│  指标风向  >  超头部成交集中度                 │
├─────────────────────────────────────────────┤
│  顶部操作栏：刷新时间 | 数据范围 | 历史窗口   │
├─────────────────────────────────────────────┤
│  集中度仪表盘（Top25 / Top1% / Top5%）       │
├─────────────────────────────────────────────┤
│  历史趋势图（三条曲线 + 分位背景）            │
├─────────────────────────────────────────────┤
│  头部股票列表（Top25 明细）                   │
├─────────────────────────────────────────────┤
│  板块集中度分布 / 排行                        │
├─────────────────────────────────────────────┤
│  资金抱团趋势解读                             │
├─────────────────────────────────────────────┤
│  行情走势变化可能性分析                       │
└─────────────────────────────────────────────┘
```

### 5.3 集中度仪表盘

- 三个卡片并排展示，移动端堆叠。
- 每个卡片包含：
  - 维度名称（Top25 / Top1% / Top5%）。
  - 当前集中度数值（百分比，保留两位小数）。
  - 较上一交易日变化值（△ 百分比，红涨绿跌或按观澜颜色规范）。
  - 当前在历史窗口中的分位标签：
    - ≥90%：极端集中
    - 75%–90%：高位集中
    - 25%–75%：正常区间
    - 10%–25%：低位分散
    - <10%：极度分散
  - 近 5 日/20 日/60 日移动平均线（可选，首期可仅显示当前值与历史均值）。

### 5.4 历史趋势图

- X 轴：交易日日期。
- Y 轴：集中度百分比。
- 三条曲线：Top25（蓝色）、Top1%（橙色）、Top5%（灰色）。
- 可点击图例隐藏/显示某条曲线。
- 支持切换窗口：20 日 / 60 日 / 120 日。
- 鼠标悬停显示当日三个维度的具体数值。
- 背景可添加分位色带（ extreme / high / normal / low ），帮助直观判断当前位置。

### 5.5 头部股票列表

- 默认展示 Top25 明细，支持按成交额、涨跌幅、市值排序。
- 列：排名、股票名称、股票代码、最新价、涨跌幅、成交额、占全市场成交额比、所属行业、主力净额（若有）。
- 点击股票名称跳转至「股票详情」页（复用现有逻辑）。
- 每只股票右侧提供「加入讨论」「加入虚拟交易」快捷入口（可选，首期可不做）。

### 5.6 板块集中度分布

- 饼图/环形图：Top25 股票成交额的行业分布。
- 排行表格：行业名称、行业成交额、占 Top25 成交额比、环比变化、代表性股票 Top3。
- 支持切换视角：Top25 / Top1% / Top5% 的板块分布。

### 5.7 资金抱团趋势解读

根据当前集中度与历史趋势，自动生成一段结构化的定性解读，至少包含：

1. **当前状态**：集中度处于历史什么分位，是否进入极端区域。
2. **趋势方向**：近 5/20/60 日是上升、下降还是震荡。
3. **板块特征**：资金主要抱团在哪些行业，是否出现板块轮动。
4. **与大盘关系**：结合当日主要指数涨跌幅，判断抱团是「逆势抱团」还是「普涨放大」。

展示形式：卡片文本 + 关键标签（如「#资金抱团加剧」「#科技线集中」「#防御板块占优」）。

### 5.8 行情走势变化可能性分析

结合规则模板与 AI 模型，输出 3～5 条可能性判断：

- **规则模板示例**：
  - 若 Top5% 集中度连续 5 日上升且处于 ≥90% 分位 → 「头部抱团处于极端状态，历史经验显示后续 5～10 个交易日出现风格扩散或抱团松动的概率较高。」
  - 若集中度从高位快速回落且大盘放量上涨 → 「资金可能从头部向中盘扩散，普涨行情概率上升。」
  - 若集中度低位横盘且市场缩量 → 「缺乏明确主线，轮动速度快，宜降低仓位或等待方向选择。」

- **AI 增强**：将当前集中度数据、历史分位、板块分布、近 5 日大盘走势、新闻政策摘要作为上下文，调用观澜理财师模型生成更自然的分析文本。AI 调用失败时，自动降级为规则模板。

- **风险提示**：在所有分析文本底部固定展示「本指标仅用于行情观察，不构成投资建议」。

### 6.3 计算任务与调度

- 页面首次加载时实时计算并返回。
- 服务端缓存：计算结果写入内存缓存，有效期 5 分钟（交易时段）/ 60 分钟（非交易时段），与现有行情缓存策略对齐。
- 日终归档：在 `jobs` 目录新增定时任务 `archiveConcentration`，于每个交易日 15:05 执行一次，将日终数据追加到 `data/concentration-history.json`。
- 自动刷新：页面在交易时段（9:15–11:30、13:00–15:00）每 5 分钟轮询一次 `/api/indicator/concentration`，与全景雷达自动刷新节奏一致。

### 6.4 性能要求

- 全市场 5000+ 股票行情获取与计算应在 10 秒内完成（含网络请求）。
- 接口响应时间 P95 ≤ 3 秒（命中缓存时）。
- 历史数据查询 ≤ 500ms。

## 10. 依赖与风险

### 10.1 依赖项

- 行情源需支持批量获取全 A 股成交额（`eastmoney` 优先）。
- 需要新增 `data/concentration-history.json` 文件用于历史归档。
- 需要新增定时任务调用现有 `node-cron` 或 `setInterval` 调度能力（复用 `src/server/jobs` 机制）。

### 10.2 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 行情源全 A 股接口限流或返回不全 | 集中度计算偏差 | 多源 fallback + 缺失样本时提示「本次样本数 N=xxxx，可能存在缺失」。 |
| 历史数据不足导致分位判断失真 | 分析质量下降 | 数据不足时隐藏分位等级，使用规则模板。 |
| AI 调用成本高或失败 | 可能性分析不可用 | 规则模板兜底，AI 作为增强。 |
| 全市场计算量大影响服务器性能 | 响应慢 | 内存缓存 + 异步归档 + 异步计算。 |
| 板块行业分类映射不全 | 板块分布不准确 | 使用东方财富分类，缺失标为「未分类」并在日志记录。 |

## 12. 附录

### 12.1 参考计算公式

```
N = 当日有效样本股票数量
k25 = 25
k1  = ceil(N × 1%)
k5  = ceil(N × 5%)

Ci(k) = Σ(amount_j for j in top k stocks by amount) / Σ(amount_i for i in all N stocks) × 100%

其中 Ci 表示维度 i ∈ {top25, top1pct, top5pct} 的集中度。
```
